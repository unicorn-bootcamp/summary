# AI 뉴스 요약 레포트 스킬 (서브 에이전트 분업 구조)

## 개요

메인 에이전트(오케스트레이터)는 **직접 작업하지 않고** 3개 서브 에이전트를 조율함.  
뉴스 5건은 상호 독립이므로 요약 작성 단계를 **건별 병렬 호출**로 전환하여 처리 시간을 단축함.

| Phase | 담당 서브 에이전트 | 팀원 | 호출 방식 |
|-------|-------------------|------|----------|
| 1 검색·원문저장·추출 | `news-researcher` | 아나(분석가) | 순차 1회 |
| 2 요약 작성 | `summary-writer` | 써미(작성가) | **병렬 5회** |
| 3 검토·조립·저장·검증 | `summary-reviewer` | 체키(검토자) | 순차 1회 |

## 오케스트레이션 절차

오케스트레이터는 아래 순서로 Agent 도구를 호출하며, 각 단계의 산출물을 다음 단계 입력으로 전달함.

### Phase 1 — 뉴스 검색·추출 (news-researcher 1회)

`news-researcher` 에이전트를 1회 호출함.
- 지시: "지난 24시간 AI 뉴스 해외 2건·국내 3건을 검색·원문 저장 후 핵심을 추출하라"
- 반환: 뉴스 5건의 구조화된 추출 마크다운(제목·URL·핵심 논점·수치·영향·구분)
- 5건 미만이거나 전부 실패 시: 사용자에게 뉴스 URL 5건 직접 입력 요청 후 재시도

### Phase 2 — 요약 작성 (summary-writer 병렬 5회)

Phase 1 추출 결과를 뉴스 건별로 분리하여 `summary-writer`를 **단일 메시지에서 5개 동시 호출**함(병렬).
- 각 호출 지시: 해당 뉴스 1건의 추출 블록을 `<뉴스데이터>...</뉴스데이터>` 태그로 임베딩하여 전달
- 각 반환: 500자 이내 유머·이모지 포함 한국어 요약 블록 1건
- 일부 호출 실패 시: 실패한 건만 재호출

### Phase 3 — 검토·조립·저장·검증 (summary-reviewer 1회)

Phase 1 추출 5건과 Phase 2 요약 초안 5건을 모아 `summary-reviewer`를 1회 호출함.
- 지시: 아래를 임베딩하여 전달
  - `<추출데이터>` : Phase 1 결과 5건
  - `<요약초안>` : Phase 2 요약 블록 5건
  - 현재 날짜·시각: `{YYYYMMDD}` / `{HHMM}`
- 검토자가 검토·교정 → 최종 레포트 조립 → `reports/news_{YYYYMMDD}_{HHMM}.md` 저장 → ls·grep 검증 수행
- 반환: 저장 경로 + 검증 증거(ls·grep 출력) + 교정 요약

### Phase 4 — 사용자 보고 (오케스트레이터)

검토자가 반환한 **검증 증거(ls·grep 출력)를 근거로** 사용자에게 완료 보고함.
- 검증 미통과(파일 미존재 또는 항목 5건 미만) 시: Phase 3 재호출, "완료" 보고 금지

## 제약조건

- 메인 에이전트는 사용자 인터랙션·조율 외 직접 작업(검색·작성·저장) 수행 금지
- 해외 뉴스는 반드시 한국어로 번역 (summary-writer 담당)
- 뉴스 1건당 500자 이내 준수
- 원문에 없는 내용 생성 금지
- 완료 조건:
  1. `reports/news_{YYYYMMDD}_{HHMM}.md` 파일이 해당 경로에 존재함
  2. 파일 내 뉴스 항목 수 = 5건 (해외 2 + 국내 3)
  3. 위 1·2를 summary-reviewer의 ls·grep 실제 출력으로 검증 완료함

## 출력 형식 (최종 레포트)

```markdown
# AI 뉴스 요약 — YYYY.MM.DD

---

### {뉴스 제목 한국어} ({YYYY.MM.DD})

**한줄요약**: {핵심을 한 문장으로}

---

{유머·이모지·비유 포함 요약본. 500자 이내.}

> 🔗 출처: {원문 URL}

---

(뉴스 2~5번 동일 형식 반복)
```
